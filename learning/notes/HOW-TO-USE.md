# How to use learning/notes/

This folder is your **personal knowledge base** — not agent-facing, but feeds into memory and genome over time.

## File naming
`YYYY-MM-DD-<topic-slug>.md`

## Structure per note
```markdown
---
date: YYYY-MM-DD
topic: <topic>
source: <url, book, experiment, conversation>
tags: []
---

# <Title>

## Key insight
One paragraph summary.

## Details
Full notes, quotes, links, diagrams.

## Questions / open threads
- ?

## Connection to project
How does this relate to current work?
```

## When to promote to memory/
When a note has been referenced 2+ times or validated through experiments, move it to `memory/semantic/`.
